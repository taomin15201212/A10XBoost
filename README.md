## 核心方法论

### 1. 工具选型四约束
- 网络与合规
- 主要开发场景
- 模型可切换性
- 验证能力

### 2. 少返工闭环
1. 先把边界说清楚（目标、约束、验收）
2. 先要"计划"，再要"代码"
3. 写完立刻要求自查
4. 报错就把"证据"喂回去

### 3. 7天落地法
每个篇章都提供 7 天落地计划，从痛点识别到可复用系统。
